# Windows Autopilot

## Contexte

La préparation des postes impliquait historiquement de nombreuses interventions manuelles.

L'objectif était d'automatiser le cycle de déploiement des équipements et de standardiser leur configuration.

---

## Objectifs

- Réduire le temps de préparation
- Standardiser les déploiements
- Améliorer l'expérience utilisateur
- Simplifier l'administration

---

## Technologies utilisées

- Windows Autopilot
- Microsoft Intune
- Microsoft Entra ID
- Microsoft Graph

---

## Réalisation

Conception et déploiement d'une architecture Windows Autopilot couvrant l'ensemble du parc informatique.

Fonctionnalités mises en œuvre :

- Entra ID Join
- User-Driven Deployment
- Pre-Provisioning
- Déploiement automatisé des applications
- Déploiement automatisé des configurations

Applications déployées :

- Microsoft 365 Apps
- Bitdefender
- Applications pédagogiques
- Outils administratifs

---

## Résultats

- Déploiement homogène des postes
- Réduction des actions manuelles
- Standardisation de l'environnement utilisateur
- Simplification du renouvellement du parc

---

## Compétences démontrées

- Windows Autopilot
- Intune
- Entra ID
- Endpoint Management
- Microsoft Graph
